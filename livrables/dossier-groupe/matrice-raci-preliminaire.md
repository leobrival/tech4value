## Légende RACI

| Code | Signification | Description |
|------|--------------|-------------|
| **R** | Responsible | Réalisateur - Celui qui fait le travail |
| **A** | Accountable | Approbateur - Celui qui valide et est responsable du résultat final |
| **C** | Consulted | Consulté - Celui dont on demande l'avis (communication bidirectionnelle) |
| **I** | Informed | Informé - Celui qui est tenu au courant (communication unidirectionnelle) |
| **A/R** | Accountable & Responsible | Cumul des rôles A et R (fréquent dans une équipe de 2 personnes) |

## Notes Importantes

### Principes de Collaboration

1. **Communication Quotidienne** : Daily standup (15 min) pour synchronisation
2. **Revue Hebdomadaire** : Point d'avancement et ajustement du planning
3. **Validation Croisée** : Chaque livrable majeur est relu par l'autre membre
4. **Flexibilité** : La matrice RACI est préliminaire et pourra être ajustée selon les besoins

### Particularités d'une Équipe de 2 Personnes

- **Cumul de rôles** : Notation A/R fréquente (un seul membre à la fois A et R)
- **Collaboration renforcée** : Les deux membres doivent être consultés sur les décisions majeures
- **Polyvalence** : Chaque membre peut aider l'autre en cas de surcharge ou de blocage
- **Charge de travail** : ~700h au total réparties équitablement (~350h par membre)

### Zones de Parallélisation Critique

**Phase 2 (Semaines 4-7)** :
- Jules : Planification + Budget + Risques (en parallèle)
- Léo : Architecture backend + frontend + intégrations (en parallèle)

**Phase 3 (Semaines 8-16)** :
- Léo : Développement des 6 fonctionnalités (en parallèle si possible)
- Jules : Préparation conduite du changement + RGPD (en parallèle)

**Phase 5 (Semaines 19-24)** :
- Jules : Sections managériales du dossier (en parallèle)
- Léo : Sections techniques du dossier (en parallèle)
- Collaboration : Vidéo + relecture finale
