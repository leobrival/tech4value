# Annexe : matrice RACI Tech4Value

## Objectif

Cette annexe précise la répartition des responsabilités entre Jules Courtin, Léo Brival, la Direction et le Comité de pilotage. Elle sert à éviter les zones grises sur les décisions, la production et la validation.

## Légende

| Code | Signification | Description |
|---|---|---|
| R | Responsible | Réalise l'action |
| A | Accountable | Valide et porte la responsabilité finale |
| C | Consulted | Donne un avis avant décision |
| I | Informed | Est tenu informé |

## Répartition par rôle

### Jules Courtin : PM / Product Manager

Responsabilités principales :

- cadrage fonctionnel ;
- priorisation des besoins ;
- planification ;
- budget ;
- risques ;
- communication Copil ;
- conduite du changement ;
- formation et adoption.

### Léo Brival : Tech Lead / Architecte Solution

Responsabilités principales :

- architecture technique ;
- développement Next.js et API ;
- modèle de données ;
- intégrations SI ;
- sécurité ;
- tests ;
- CI/CD ;
- documentation technique.

### Direction

Responsabilités principales :

- validation des priorités stratégiques ;
- arbitrage budgétaire ;
- validation des jalons majeurs ;
- sponsoring du changement.

### Comité de pilotage

Responsabilités principales :

- suivi de l'avancement ;
- validation des jalons ;
- arbitrage des risques ;
- coordination des parties prenantes.

## Principes de fonctionnement

- Toute décision structurante est documentée.
- Jules porte la valeur métier ; Léo porte la cohérence technique.
- Les arbitrages de périmètre sont faits en commun.
- Les blocages critiques sont remontés en Copil.
- Les utilisateurs pilotes sont consultés avant généralisation.

## Synthèse

La matrice RACI confirme que le projet est porté par une équipe interne réduite mais structurée. La clarté des rôles permet de limiter les doublons, d'accélérer les décisions et de sécuriser la livraison du MVP.
