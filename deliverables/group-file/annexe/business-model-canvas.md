# Annexe : Business Model Canvas Tech4Value

## Objectif

Cette annexe résume le modèle de valeur de Tech4Value. Elle montre comment la plateforme répond au besoin interne de pilotage stratégique et comment elle pourrait évoluer vers une offre SaaS pour des PME ou ETI confrontées aux mêmes problèmes.

## Canvas synthétique

| Bloc | Synthèse |
|---|---|
| Segments clients | Direction, PMO, DSI, chefs de projet ; à terme PME/ETI de 50 à 500 collaborateurs |
| Proposition de valeur | Une plateforme unique pour piloter les projets, réduire le reporting manuel et fiabiliser les décisions |
| Canaux | Déploiement interne, démonstrations Copil, formation utilisateurs ; à terme ventes directes et partenaires SI |
| Relation client | Accompagnement au déploiement, support, formation, amélioration continue |
| Revenus | Valeur interne mesurée par gains de productivité ; à terme abonnement SaaS et services d'intégration |
| Ressources clés | Stack TypeScript, connecteurs Odoo/SharePoint/Power BI, expertise PMO, données projet consolidées |
| Activités clés | Développement produit, synchronisation SI, reporting, support, conduite du changement |
| Partenaires clés | Microsoft, Odoo, hébergeurs cloud, intégrateurs SI, équipes internes Tech4Value |
| Structure de coûts | Effort interne, infrastructure cloud, maintenance, support, formation, évolutions produit |

## Segments utilisateurs internes

| Segment | Besoin principal | Valeur apportée |
|---|---|---|
| Direction | Vision consolidée et arbitrages rapides | Dashboard portefeuille et indicateurs comparables |
| PMO | Réduire la consolidation manuelle | Reporting automatisé et source de données unique |
| Chefs de projet | Suivre avancement, risques et ressources | Vue projet claire et mise à jour régulière |
| DSI | Maîtriser l'intégration au SI | Architecture documentée, sécurité, SSO |
| Fonctions support | Accès aux données utiles | Consultation ciblée selon les droits |

## Proposition de valeur

Tech4Value apporte une réponse à trois irritants majeurs :

1. fragmentation des outils de suivi ;
2. reporting manuel chronophage ;
3. décisions prises sur des données trop anciennes.

La promesse centrale :

> Un seul écran pour piloter tous les projets stratégiques.

## Revenus et valeur économique

Dans le contexte interne, la valeur ne vient pas d'un chiffre d'affaires direct mais des gains opérationnels :

- réduction du temps de reporting PMO ;
- meilleure allocation des ressources ;
- réduction des retards ;
- diminution des doublons de saisie ;
- décisions plus rapides en Copil.

Projection externe possible :

| Offre | Description |
|---|---|
| Abonnement SaaS | Accès plateforme par organisation |
| Implémentation | Paramétrage, intégrations, reprise de données |
| Formation | Sessions utilisateurs et administrateurs |
| Support | Assistance et maintenance évolutive |

## Coûts principaux

| Poste | Commentaire |
|---|---|
| Ressources humaines | Effort interne Jules / Léo pour le MVP |
| Infrastructure | Hébergement frontend, API et base de données |
| Licences | Services cloud et outils de productivité |
| Support | Assistance utilisateurs et maintenance |
| Évolutions | Nouvelles intégrations, forecasting, mobile |

## Risques business

| Risque | Impact | Réponse |
|---|---|---|
| Adoption insuffisante | Valeur réduite | Formation, ambassadeurs, mesure d'usage |
| Données sources incomplètes | Reporting peu fiable | Contrôles qualité et fallback |
| Périmètre trop large | Retard MVP | Priorisation MoSCoW |
| Coûts de production sous-estimés | ROI fragilisé | Suivi mensuel et scénario progressif |

## Synthèse

Le modèle économique est d'abord un modèle de valeur interne. La plateforme se justifie par le temps gagné, la fiabilité des données et l'amélioration du pilotage. L'ouverture SaaS reste une option long terme, crédible uniquement après validation du MVP interne.
