# Annexe : roadmap Now-Next-Later Tech4Value

## Objectif

Cette annexe présente la roadmap produit du MVP Tech4Value selon le framework Now-Next-Later. Elle permet de distinguer ce qui est prioritaire maintenant, ce qui vient ensuite et ce qui reste une option d'évolution.

## Principe

Le framework Now-Next-Later évite une roadmap figée. Il classe les initiatives par horizon :

| Horizon | Sens | Décision associée |
|---|---|---|
| Now | À faire ou en cours | Engagement fort |
| Next | À préparer | Priorisation probable |
| Later | À explorer | Hypothèse non engagée |

## Now : MVP interne

Objectif : livrer une plateforme utile pour piloter les 5 projets stratégiques.

| Initiative | Résultat attendu | Owner |
|---|---|---|
| Cadrage fonctionnel | Besoin priorisé et validé | Jules |
| Architecture trois-tiers | Socle technique maintenable | Léo |
| Dashboard portefeuille | Vision consolidée des projets | Léo |
| Synchronisation Odoo | Données projet et budget intégrées | Léo |
| RACI et gouvernance | Responsabilités clarifiées | Jules |
| Reporting Copil | Réduction du reporting manuel | Jules + Léo |
| Formation pilotes | Adoption initiale maîtrisée | Jules |

## Next : consolidation

Objectif : stabiliser le MVP, améliorer la qualité et préparer un usage élargi.

| Initiative | Résultat attendu |
|---|---|
| Tests E2E renforcés | Parcours critiques sécurisés |
| Optimisation performance | Latence réduite sur dashboards |
| Export Power BI | Restitution avancée pour utilisateurs experts |
| Documentation utilisateur | Prise en main autonome |
| Mesure adoption J+30/J+90 | Pilotage de la conduite du changement |
| Amélioration UX | Parcours plus simples par rôle |

## Later : évolution produit

Objectif : préparer les options de montée en puissance sans surcharger le MVP.

| Option | Intérêt |
|---|---|
| Application mobile | Consultation rapide des KPI |
| Forecasting IA | Anticipation des retards et dépassements |
| Connecteur Power BI natif | Analyse décisionnelle plus fluide |
| API partenaires | Ouverture à d'autres outils SI |
| Offre SaaS | Valorisation externe pour PME/ETI |

## Jalons principaux

| Jalon | Horizon | Résultat attendu |
|---|---|---|
| M1 | S4 | Cadrage et périmètre validés |
| M2 | S6 | Environnement technique opérationnel |
| M3 | S8 | Authentification et dashboard de base |
| M4 | S12 | Intégrations Odoo et Azure AD stabilisées |
| M5 | S14 | MVP complet sur fonctionnalités Must Have |
| M6 | S18 | Tests et performance validés |

| M8 | S24 | Mise en production interne et bilan |

## Priorisation

Les fonctionnalités sont arbitrées selon quatre critères :

| Critère | Question |
|---|---|
| Impact | Quel gain pour le PMO ou la direction ? |
| Effort | Quel volume de travail pour Jules et Léo ? |
| Risque | Quelle incertitude technique ou métier ? |
| Dépendance | L'initiative bloque-t-elle d'autres travaux ? |

## Synthèse

La roadmap protège le MVP contre le scope creep. Les fonctionnalités directement liées à la réduction du reporting et à la consolidation des données sont traitées en priorité. Les évolutions plus ambitieuses restent identifiées, mais ne compromettent pas la livraison interne.
