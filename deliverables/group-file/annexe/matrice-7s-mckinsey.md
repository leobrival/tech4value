# Annexe : matrice McKinsey 7S Tech4Value

## Objectif

Cette annexe synthétise l'analyse organisationnelle Tech4Value selon le modèle McKinsey 7S. Elle identifie les dimensions déjà alignées et les points à accompagner pour réussir l'adoption de la plateforme.

## Vue d'ensemble

Le modèle 7S analyse sept dimensions interdépendantes :

| Dimension | Question clé |
|---|---|
| Strategy | La stratégie soutient-elle le projet ? |
| Structure | L'organisation facilite-t-elle le pilotage transverse ? |
| Systems | Les systèmes et processus sont-ils adaptés ? |
| Shared Values | Les valeurs favorisent-elles la transparence et la décision par la donnée ? |
| Style | Le style de management encourage-t-il l'arbitrage factuel ? |
| Staff | Les équipes disposent-elles des ressources nécessaires ? |
| Skills | Les compétences requises sont-elles maîtrisées ? |

## Diagnostic synthétique

| Dimension | Situation actuelle | Cible avec Tech4Value | Niveau de risque |
|---|---|---|---|
| Strategy | Volonté de mieux piloter les projets stratégiques | Pilotage portefeuille data-driven | Faible |
| Structure | Organisation matricielle avec 9 pôles | Coordination renforcée PMO / pôles | Moyen |
| Systems | 7 outils dispersés, reporting manuel | Plateforme consolidée + intégrations SI | Élevé |
| Shared Values | Culture projet existante mais données fragmentées | Transparence et source unique de vérité | Moyen |
| Style | Arbitrages encore dépendants de réunions et fichiers | Décision appuyée sur KPI communs | Moyen |
| Staff | 185 collaborateurs, 78 ETP engagés sur 5 projets | Rôles clarifiés et adoption progressive | Moyen |
| Skills | Maturité digitale hétérogène selon les pôles | Formation reporting, dashboard et nouveaux rituels | Élevé |

## Points forts

- La direction a un besoin clair de visibilité consolidée.
- Le PMO est légitime pour porter la gouvernance portefeuille.
- Les briques SI principales existent déjà : Odoo, SharePoint, Power BI, Azure AD.
- Le périmètre MVP est compatible avec une équipe interne réduite.

## Points de vigilance

- Les habitudes Excel et outils locaux peuvent freiner l'adoption.
- Les managers intermédiaires doivent être associés tôt.
- La qualité des données sources conditionne la crédibilité du dashboard.
- La conduite du changement doit être traitée comme un volet projet à part entière.

## Plan d'action associé

| Action | Dimension 7S | Responsable |
|---|---|---|
| Clarifier les rôles RACI | Structure / Staff | Jules |
| Standardiser les KPI projet | Systems / Strategy | Jules + Léo |
| Déployer les formations par profil | Skills / Staff | Jules |
| Sécuriser les intégrations SI | Systems | Léo |
| Créer un réseau d'ambassadeurs | Shared Values / Style | Jules |
| Suivre l'adoption à J+30 et J+90 | Systems / Style | PMO |

## Lecture globale

Les dimensions les plus alignées sont **Strategy** et **Structure** : le besoin stratégique existe et l'organisation dispose déjà d'un PMO. Les dimensions les plus sensibles sont **Systems** et **Skills**, car elles nécessitent à la fois une plateforme fiable et une montée en compétence des utilisateurs.

## Synthèse

Le projet Tech4Value n'est pas uniquement technique. Sa réussite dépend de l'alignement entre outils, rôles, pratiques de management et compétences. La matrice 7S confirme que la solution doit être accompagnée par une conduite du changement structurée.
